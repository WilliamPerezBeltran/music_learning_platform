# Roadmap MVP Music Learning Platform

## Fase 0 — Validación Técnica

### Objetivo

Validar que el stack tecnológico puede soportar el producto.

### Entregables

* Proyecto Phoenix + LiveView inicial.
* Integración OpenSheetMusicDisplay.
* Integración Tone.js.
* Renderizar un archivo MusicXML.
* Reproducir una canción simple.
* Resaltar notas durante la reproducción.

### Resultado esperado

Demostración funcional con una única canción.

# Fase 2 — Sistema de Niveles

### Objetivo

Crear progresión educativa.

### Funcionalidades

Cada canción tendrá:

```text
Nivel 1 - Melodía
Nivel 2 - Melodía simplificada
Nivel 3 - Acordes básicos
Nivel 4 - Dos manos simplificadas
Nivel 5 - Canción completa
```

### Administración

* Asociar múltiples niveles a una canción.
* Ordenar niveles.
* Publicar/despublicar niveles.

### Resultado esperado

Usuario avanza progresivamente dentro de una misma canción.

# Hito de Validación

El producto se considera validado cuando:

* Existen usuarios activos recurrentes.
* Los usuarios completan canciones.
* Existen usuarios que pagan.
* El contenido genera tráfico orgánico.
* El sistema de colores demuestra acelerar el aprendizaje.

Hasta alcanzar este punto no se debe invertir tiempo significativo en IA, aplicaciones nativas o funcionalidades complejas.
